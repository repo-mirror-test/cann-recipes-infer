### YAML Parameter Description

The configuration instructions in the YAML file can be found below. We are given a YAML configuration file and a description that mentions two variables DATE and CASE_NAME which are automatically generated by a script `executor/scripts/function.sh`.The `CASE_NAME` is formed by concatenating the `model_name` with the YAML file name.The YAML content is:

```yaml
model_name: "gpt_oss_20b"        # Model name
model_path: "your_model_path"    # Path to the model weights
exe_mode: "eager"                # ["eager"], mode of decode
world_size: 1                    # Number of chips used for inference

model_config:
  enable_profiler: False         # [False, True] Whether to enable profiling; default cache path is `./res/DATE/CASE_NAME`
  enable_online_split_weight: True # Whether to enable online weight splitting

data_config:
  dataset: "LongBench"           # ["default", "LongBench"] The content of the input prompt
  input_max_len: 128             # Input length of the request
  max_new_tokens: 100            # Maximum output length
  batch_size: 1                  # Total number of requests globally

parallel_config:
  attn_tp_size: 1                # Attention tensor parallelism size(must equal to moe_tp_size and lmhead_tp_size)
  moe_tp_size: 1                 # MoE tensor parallelism size
  lmhead_tp_size: 1              # lmhead tensor parallelism size
```
